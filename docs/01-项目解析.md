# WaiChat 项目解析

## 1. 项目结构

本仓库当前由两个子项目组成：

- `WaiChat-master`：后端服务（Spring Boot 3 + MyBatis-Plus + Redis + WebSocket + AI 能力）
- `WaiChat-vue`：前端应用（Vue 3 + Vite + Axios + ECharts）

前后端在开发环境通过 Vite 代理联调：

- `/api` -> `http://127.0.0.1:8080`
- `/ws` -> `ws://127.0.0.1:8080/ws`
- `/uploads` -> `http://127.0.0.1:8080/uploads`

## 2. 技术栈

### 后端（`WaiChat-master`）

- Java 17
- Spring Boot 3.5.x
- Spring Security（表单登录，`/login`）
- Spring WebSocket（端点：`/ws/{userId}`）
- MyBatis-Plus（MySQL 持久化）
- Redis（会话热数据/聊天缓存）
- DashScope SDK（翻译、语音转文字、润色、智能回复、总结、分析）

### 前端（`WaiChat-vue`）

- Vue 3 + Vue Router
- Vite 7
- Axios
- Pinia（当前使用较少）
- ECharts（聊天分析看板）

## 3. 核心业务能力

- 用户注册、登录
- 联系人搜索
- 单聊实时消息（WebSocket + HTTP 兜底）
- 聊天历史查询、隐藏与恢复
- AI 辅助：
  - 多语言翻译
  - 语音转文字（URL 或文件）
  - 文本润色
  - 智能回复建议
  - 聊天总结
  - 聊天分析（关键词、情感、总结）

## 4. 运行链路（简述）

1. 前端登录后保存 `userId/username/nickname` 到本地。
2. 进入聊天页后建立 WebSocket 连接 `/ws/{userId}`。
3. 发消息优先走 WebSocket，后端同步写 Redis，异步落库 MySQL，并推送给目标用户。
4. 历史查询和联系人列表通过 REST API 拉取。
5. AI 功能由前端调用 `/api/ai/*` 接口，后端转发到 DashScope 模型服务。

## 5. 安全与运维关注点

- 目前 `application.yml` 中存在数据库、Redis、AI Key 等敏感配置，建议改为环境变量注入。
- Spring Security 当前放行 `/login`、`/register`、`/api/chat/send`、`/uploads/**`，需按生产需求评估最小权限。
- WebSocket 连接参数仅依赖 `userId`，生产建议叠加 token 校验防止伪造连接。
- 文件上传路径基于本地目录，生产建议接入对象存储并增加类型/大小校验。
