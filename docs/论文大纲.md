# 毕业论文大纲

**论文题目（建议）**：基于 AI 的跨语言即时通信与翻译系统的设计与实现  

**说明**：本大纲对应仓库项目 **WaiChat**（Vue3 前端 + SpringBoot 后端 + WebSocket + MySQL/Redis + 大模型 API）。正文撰写时可在各节下继续拆三级标题；摘要、关键词、参考文献、致谢、附录按学院模板单独编排。

---

## 1 绪论

### 1.1 研究目的与意义

- 跨语言沟通与 Web 即时协作的需求背景  
- 独立翻译工具与即时通信割裂带来的问题（上下文迁移、延迟、协议开销等）  
- 本课题目标：集成实时通信与智能翻译的 Web 系统及其理论与实践意义  
- 与教学要求、轻量应用场景的契合说明  

### 1.2 国内外研究现状

- Web 即时通信技术路线（轮询、SSE、WebSocket 等）及演进  
- 翻译技术代际与「对话 + 翻译」一体化应用的研究与产品形态  
- 与本课题的定位差异（单聊、托管大模型、教学级实现深度等）  

---

## 2 相关知识

### 2.1 Web 实时通信与 WebSocket

- HTTP 请求—响应模型与「服务端推送」需求  
- WebSocket：握手升级、全双工、与 HTTP 的关系（概念层面）  
- 与轮询、SSE 的对比及本文选用 WebSocket 的理由  
- 工程要点：连接与用户绑定、断线重连、报文格式、与业务推送的关系  

### 2.2 机器翻译至人工智能翻译的技术演进

- 规则与词典方法及其局限  
- 统计机器翻译（SMT）与数据驱动思路  
- 神经机器翻译（NMT）、注意力与 Transformer  
- 大语言模型时代的「翻译即生成」、上下文与可控性、幻觉与成本约束  
- 与本文关系：不重复训练基座模型，侧重会话场景下的集成  

### 2.3 大模型 API 调用

- 托管式生成接口的一般形态（同步/异步、鉴权、配额与超时）  
- 请求与响应封装（DTO/VO、统一错误包络）  
- 与本系统 DashScope（或同类）SDK 的接入方式及安全注意点  
- 与会话链路的解耦：避免阻塞实时推送线程的设计原则

## 3 系统设计

### 3.1 系统目标与功能边界

- 功能范围：注册登录、单聊、联系人、文本/语音、历史隐藏与恢复、翻译与 AI 辅助等  
- 非目标：大规模群聊、端到端加密、完整运维平台等（如实界定）  

### 3.2 总体架构

- 浏览器端 / 应用服务 / 数据层 / 外部大模型服务的逻辑分层  
- 逻辑结构图或部署示意图（可与第 5 章成果图呼应）  

### 3.3 技术选型及依据

- Vue3、Vite、SpringBoot、WebSocket、MySQL、Redis、大模型 API 等及其取舍  

### 3.4 核心业务流程

- 身份建立 → 长连接 → 消息发送与推送 → 持久化 → 历史查询  
- 文本与语音的「双通道」互补设计（同步接口与长连接分工）  

### 3.5 关键机制设计要点

- 统一点对点发送、缓存与异步落库、身份覆盖与防伪造思路  
- 接口层与领域层边界（可结合时序图）
